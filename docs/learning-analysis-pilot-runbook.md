# 학습 증거 분석 파일럿 운영 절차

이 기능은 LMS와 Grade App이 같은 Supabase 프로젝트를 사용하는 것을 전제로 한다. Grade App 코드만 먼저 배포하면 `learning.submit_session_v2`가 없어 제출이 실패하므로 다음 순서를 지킨다.

## 1. 배포 순서

1. 데이터베이스 백업과 기존 중복 사전 검사를 실행한다.
2. LMS의 다음 migration을 순서대로 적용한다.
   - `20260710172114_learning_evidence_schema_v1.sql`
   - `20260710172924_atomic_learning_submission_v2.sql`
   - `20260710174907_create_analysis_plan_v1.sql`
3. LMS를 배포한다.
4. Grade App을 배포한다.
5. 시험용 학생 계정으로 과제 1건을 제출하고 LMS 근거 화면에서 확인한다.

기존 DB 사전 검사는 migration 자체에서도 수행한다. 학생·과제 중복 세션, 세션 내 중복 문항, 중복 회차, 잘못된 음수 시간이 있으면 migration은 조용히 합치지 않고 중단한다.

## 2. 파일럿 taxonomy 준비

기본 명령은 읽기 전용이다.

```bash
npm run db:bootstrap-learning-analysis -- --book-id <BOOK_UUID>
```

요약을 검토한 뒤 draft와 pending tag를 만든다.

```bash
npm run db:bootstrap-learning-analysis -- --book-id <BOOK_UUID> --apply
```

난이도와 유형 매핑을 검토했다면 게시·승인한다. `difficulty_hint`가 정확히 `하/중/상/최상`인 문항만 자동으로 1~4 단계에 대응한다. 다른 값은 pending으로 남는다.

```bash
npm run db:bootstrap-learning-analysis -- --book-id <BOOK_UUID> --apply --approve
```

레거시 데이터에는 동일·유사 문항 관계가 없으므로 bootstrap은 각 `problem_id`를 서로 다른 equivalence key로 가정한다. 이는 파일럿 편의를 위한 보수적이지 않은 가정이다. 반복 변형이나 사실상 같은 문항이 있으면 승인 전 동일 key로 묶어야 한다.

## 3. 강사 사용 흐름

1. LMS `학습 분석`에서 현행·선행·유지 복습 또는 시험 계획을 만든다. 유지 복습 계획은 교재 없이도 저장할 수 있다.
2. Grade App에서 평소처럼 채점한다.
3. LMS의 통합 조치 목록에서 학생·유형을 선택하고 `과제 초안 만들기`를 누른다.
   - 학생마다 필요한 유형 조합이 다르면 한 과제로 합치지 않고 같은 조합끼리 나눈다.
4. 과제 관리 화면에서 미리 선택된 학생과 유형 안내를 확인하고 문제집·문제 범위를 직접 검토한다.
5. `배포`를 누르기 전에는 학생에게 과제가 전송되지 않는다.

## 4. 해석 원칙

- 최초 독립 오답은 수정하지 않는다.
- 즉시 교정, drill, 도움받은 풀이는 확인 완료 근거에서 제외한다.
- 미응답은 오답이 아니며 실력 분석에서 제외한다.
- `모름`은 명시적인 부정 근거다.
- 배정 범위는 서버가 확정한다. 일부 문항을 요청에서 빼도 부분 제출로 확정되지 않는다.
- 최근 확인과 지원 후보는 서로 다른 날짜·문제·equivalence key의 독립 근거 2회가 필요하다.
- 같은 날 정답과 오답이 함께 있으면 혼합 근거로 보고 성급하게 지원 후보로 올리지 않는다.
- 21일 등 계획 주기가 지나도 학생을 미달로 바꾸지 않는다. 재확인 표시를 유지하고, 주기 이후 새로운 독립 근거 2회가 생기면 확인일을 갱신한다.
- 자료 부족은 학생 상태가 아니라 콘텐츠 관리 항목이다.
- taxonomy 개정 뒤에도 이전 계획은 당시 revision의 문제 태그와 근거를 그대로 사용한다.
- 시험 계획은 시험일까지 활성 범위에 남고, 지난 시험은 통합 조치 큐에서 자동으로 제외한다.
- 분석 조치로 과제를 배포하면 같은 학생·유형의 중복 조치를 숨기고, 제출 또는 새 독립 근거가 생기면 다시 판정한다.

## 5. 파일럿 점검 항목

- 교재 1권, 1~2개 반, 시험 계획 1개로 시작한다.
- 시스템의 `확인 필요`와 `지원 후보`를 강사의 실제 판단과 비교한다.
- 문항·유형·도전 단계 오류로 생긴 오판을 별도로 기록한다.
- 자동 배포는 하지 않는다.
- 학생·학부모 공개, 학생 순위, 강사 평가에는 사용하지 않는다.
